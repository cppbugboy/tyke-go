## 核心模块

### 1. Framework

框架核心，管理整个应用的生命周期。

**职责：**
- 初始化日志系统
- 初始化 WorkerPool
- 启动 IPC 服务器
- 管理控制器注册

**设计模式：** 单例模式（通过 `sync.Once` 实现）

### 2. IpcServer

IPC 服务器，处理进程间通信。

**职责：**
- 接受客户端连接
- 数据加密/解密
- 消息分发

**设计模式：** PIMPL 模式（隐藏平台实现）

### 3. WorkerPool

协程池，管理异步任务执行。

**职责：**
- 任务队列管理
- Worker 管理
- 优雅退出

**设计模式：** 单例模式

### 4. Router

路由器，管理请求/响应处理。

**职责：**
- 路由注册
- 过滤器链
- 处理器分发

## 协议设计

### 协议头（28 字节）

| 字段 | 大小 | 说明 |
|------|------|------|
| Magic | 4B | 魔数 "TYKE" |
| MsgType | 4B | 消息类型 |
| Reserved | 12B | 保留字段 |
| MetadataLen | 4B | 元数据长度 |
| ContentLen | 4B | 内容长度 |

## 加密机制

### 握手流程

```
Client                          Server
  │                               │
  │──── ECDH Public Key ────────>│
  │                               │
  │<──── ECDH Public Key ────────│
  │                               │
  │     Compute Shared Secret    │
  │                               │
  │<──── Encrypted Data ────────>│
```

### 加密算法

- **密钥交换**：ECDH P-256
- **对称加密**：AES-256-GCM
- **密钥派生**：SHA-256

## 并发模型

### Goroutine 使用

- **Accept Loop**：独立 goroutine 接受连接
- **Handle Client**：每个客户端一个 goroutine
- **Worker Pool**：固定数量的 worker goroutine

### 同步机制

- `sync.Mutex`：保护共享资源
- `sync.Pool`：对象池
- `sync.Once`：单例初始化
- `sync.WaitGroup`：等待组

## 内存管理

### 对象池

- `TykeRequest` 和 `TykeResponse` 使用 `sync.Pool`
- 减少内存分配开销

### 数据复制

- 异步处理时复制数据，避免数据竞争
- 使用 `copy()` 函数复制字节切片

## 日志系统

### 日志级别

- Debug
- Info
- Warn
- Error

### 日志轮转

使用 `lumberjack` 实现日志文件轮转：

```go
InitLog(logPath, logLevel, fileSizeMB, fileCount)
```

### 结构化日志

基于 `slog` 实现结构化日志：

```go
LogInfo("Request received, route=%s, uuid=%s", route, uuid)
```
